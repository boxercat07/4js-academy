17. Quizzes & Content Locking

Interactive quizzes generated from NotebookLM JSON exports. Supports sequential locking where subsequent module items are blocked until the quiz is passed.

Content Locking States

Rule: Items are automatically locked if they follow a QUIZ that has not been passed (score < success-threshold).

Interactive Quiz Component

<ai-quiz title="Module 01 Quiz" src="path/to/notebooklm-export.json" success-threshold="80"> </ai-quiz>

Quiz Attributes

Attribute Type Description
title String Quiz name (also used as localStorage key)
src URL Path to a NotebookLM JSON export or blob URL
success-threshold Number (0–100) Minimum % to pass. Default: 80

Quiz Features

  • Randomized Options — Answer order is shuffled for each attempt
  • Instant Feedback — Correct/Incorrect shown after each answer
  • Best Score Persistence — Only the highest score is saved in localStorage
  • Exit Confirmation — A modal prevents accidental progress loss
  • Module Locking — Content below an unpassed quiz is locked
  • Flexible JSON — Supports options / choices , boolean flags, string/index answers
